Module behaviour

Behaviour module

This module contains methods to collect and send user behaviour data to AimBrain API.

View identifiers

Assigning view identifiers to the views improves accuracy of analysis. Use setAimbrainId to set identifier to a DOM element.

Usage example:

var element = document.getElementById('an-element');
Aimbrain.behaviour.setAimbrainId('your-id', element);

Same effect can be achieved by setting html attribute data-aimbrain-id. The SDK will use HTML attribute id if no custom id is set and id.

<div data-aimbrain-id="custom"/> <!-- SDK will use id 'custom' -->
<div data-aimbrain-id="custom" id="an-id"/> <!-- SDK will use id 'custom' (aimbrain-id attribute takes precedence) -->
<div id="an-id"/> <!-- SDK will use id 'an-id' (use id attribute)  -->
<div class="a-class"/> <!-- SDK will use no id -->

Starting and stopping behavioural data collection

Call startTracking to begin behavioural event collection. stopTracking stops tracking.

Usage example:

Aimbrain.behaviour.startTracking();

Collected events are stored in local storage until they are sent to AimBrain servers and persist on page reloads.

Tracking starts only after call to SDK method and stops after page reload. startTracking needs to be called on each page reload to keep behavioural tracking.

Submitting behavioural data

Call sendCollectedEvents to send collected data to AimBrain for processing.

Calling this method with parameter serialized set to true returns promise resolving to string with serialized request JSON:

Aimbrain.behaviour.sendCollectedEvents(true).then(jsonString => {
     // use request body in jsonString
    console.log(jsonString);
});

Periodic behavioural data submission

Call to startSending begins periodic submission of collected data and stopSending stops background sending if it was started.

Use convenience method start to begin behavioural tracking and periodic data submission using default value:

Aimbrain.behaviour.start(5000);

Use interval parameter for start and startSending to control sending frequency.

Getting current score

To get the current session score from the server without sending any data use getScore:

var score = Aimbrain.behaviour.getScore();

Limiting sensitive data capture

Sensitive view data capture limiting is achieved by calling the addToSensitiveList method with DOM element or with an array of DOM elements to be treated as sensitive views. Events captured on sensitive views do not contain absolute click location and the view identifiers are salted using device specific random key.

Same effect can be achieved by setting html attribute data-aimbrain-sensitive

<div data-aimbrain-sensitive="true"> ...

Adding DOM element to sensitive list makes all its children sensitive as well.

Use removeFromSensitiveList to remove elements from sensitive list.

Sensitive element list does not persist across page reloads. Elements need to be added on each page reload.

Disabling sensitive data capture

Calling to addToUntrackableList disables tracking of DOM element(s). Absolutely no data will be collected about user activity connected with the untrackable view.

Same effect can be achieved by setting html attribute data-aimbrain-ignored

<div data-aimbrain-ignored="true"> ...

Adding DOM element to untrackable list makes all its children untrackable as well.

Use removeFromUntrackableList to remove elements from untracked list.

Untracked element list does not persist across page reloads. Elements need to be added on each page reload.

Getting the current score

To get the latest session score received from server use getScore. Calling this method does not make any network API calls.

var score = Aimbrain.behaviour.getScore();
