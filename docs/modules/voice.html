Module voice

Voice module

Voice module provides functions to record audio of user voice and use recorded audio for enrollment or authentication.

Retrieving voice token

Voice token is a text retrieved via SDK and presented to the user to be read.

Use getAuthenticationToken to get voice token for authentication.

Aimbrain.voice.getAuthenticationToken().then((result) => {
    // present result to user for reading
});

Use getEnrollmentToken to get enrollment token. getEnrollmentToken takes token key which identifies each enrollment step. Use getEnrollmentTokenKeys to get all keys for enrollment. User needs to successfully enroll in all steps to finish enrollment.

var tokenKeys = Aimbrain.voice.getEnrollmentTokenKeys();

/* for each value in tokenKeys: */
var currentStepKey = tokenKeys[...];
Aimbrain.voice.getEnrollmentToken(currentStepKey).then((voiceToken) => {
    // voice token now contains token to be presented
    Aimbrain.voice.recordVoice(currentStepKey).then((voiceRecord) => {
        // voiceRecord is voice recorded
        Aimbrain.voice.enrollWithData(voiceRecord).then((result) => {
            // current step is succesfull, use next token
        });
    });
});

Recording user voice

Use recordVoice to display recording popup to record user voice.

Aimbrain.voice.recordVoice().then((result) => {
    // result contains base64 encoded voice data recorded
}).catch((error) => {
    // voice recording failed, use error code find out why.
    switch (error) {
        case Aimbrain.voice.RecordingError.PopupClosed:
        // popup closed by user
        break;
        case Aimbrain.voice.RecordingError.NotSupported:
        // browser does not support recording
        break;
        case Aimbrain.voice.RecordingError.NoDevice:
        // there is no recording device present
        break;
    }
});

Customizing recording dialog

Use methods [[setTopLabel]] and [[setBottomLabel]] to set text on popup. [[setRecordningLabel]] and [[setTimerLabel]] to control recording countdown label.

These methods must be invoked before calling recordVoice and set labels persist until page reload.

Presented popup has the following structure.

<div class="aimbrain-dialog-wrapper">
    <div class="aimbrain-dialog-voice">
        <div class="aimbrain-voice-close">
            <!-- contains close button -->
        </div>
        <div class="aimbrain-voice-header-label">
            <!-- contains top text -->
        </div>
        <div class="aimbrain-voice-token">
            <!-- contains text to be read -->
        </div>
        <div class="aimbrain-voice-capturing-label">
            <!-- contains capture progress text -->
        </div>
        <div class="aimbrain-voice-controls">
            <div class="aimbrain-voice-progress-label">
                <!-- contains capture time progress -->
            </div>
            <div class="aimbrain-voice-record-button">
                <!-- contains recording button -->
            </div>
            <div class="aimbrain-voice-stop-button">
                <!-- contains recording stop button -->
            </div>
        </div>
    </div>
</div>

Element classes will be always present and can be used to amend popup styling with custom CSS.

Enrolling with the voice module

Use enrollWithData to enroll with recorded voice. Data received from recordVoice should be passed as enrollWithData argument without modifications.

Aimbrain.voice.recordVoice(options).then((voiceToken) => {
    Aimbrain.voice.enrollWithData(result).then((result) => {
        // voice step enrolled, reestablish session to get latest enrollment state.
    }).catch((error) => {
        // handle enroll error
    });
}).catch((error) => {
    // handle record error
});

Calling this method with parameter serialized set to true returns promise resolving to string with serialized request JSON:

Aimbrain.voice.enrollWithData(voiceData, true).then(jsonString => {
    console.log(jsonString);
});

Authenticating with the voice module

Use authenticateWithData to authenticate with recorded voice. Data received from recordVoice should be passed as authenticateWithData argument without modifications.

Aimbrain.voice.recordVoice(options).then((result) => {
    Aimbrain.voice.authenticateWithData(result).then((result) => {
        // use result.score and result.liveliness fields
        // to decide authentication outcome
    }).catch((error) => {
        // handle enroll error
    });
}).catch((error) => {
    // handle record error
});

Calling this method with parameter serialized set to true returns promise resolving to string with serialized request JSON:

Aimbrain.voice.authenticateWithData(voiceData, true).then(jsonString => {
    // use request body in jsonString
    console.log(jsonString);
});
