Module session

Session module

In order to use Behaviour, Facial and Voice modules you need to have an active session. Depending on use case you can obtain new session or set existing session created via server side API.

Once set the session is persisted in local storage until clearSession method is called.

Accessing current session status

Use isSessionActive method to determine if there is an active session established. The module contains methods such as getSessionId, isFacialEnrolled, isVoiceEnrolled to get current session state.

Obtaining new session

When accessing AimBrain API servers via SDK use establishSession to create a new session. Use isSessionActive to check for existing session to avoid unnecessary session creation.

Session initialization example:

if (Aimbrain.session.isSessionActive()) {
    // session already exists, use sdk functionality
    ...
}
else {
    Aimbrain.session.establishSession("[user id]").then((result) => {
        // use sdk functionality if session create result is succesfull
        ...
    });
}

Using existing session

User and session identifiers can be set directly with setExistingSession. This method sets identifiers without calls to API server and is meant to be used with request serialization.
