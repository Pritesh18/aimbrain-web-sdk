Behavioural tracking tests

Before each test:
  • Clear session
  • Open dev tools with network request inspection on

#behavioural1 - module support

Test steps:

  • Refresh session state
Expected result:
  • Module is supported if browser supports Indexed DB
  • If module is not supported, n/a tests #4 - #7

#behavioural2 - session persistence

Test steps:

  • Clear session
  • Establish session
  • Refresh page
  • Refresh session state
Expected result:
  • Session id is filled after establish
  • Session id is filled after refresh

#behavioural3 - session clear

Test steps:

  • Establish session
  • Clear session
Expected result:
  • Session id is filled after establish
  • Session id and score are n/a after clear

#behavioural4 - send events

Test steps:

  • Establish session
  • Start tracking
  • Move and click mouse
  • Stop tracking
  • Send events
Expected result:
  • Event upload request is submitted with collected event data
  • Refreshing session state shows score as a number

#behavioural5 - send events page reload

Test steps:

  • Establish session
  • Start tracking
  • Move and click mouse
  • Stop tracking
  • Reload page
  • Send events
Expected result:
  • Event upload request is submitted with collected event data
  • Refreshing session state shows score as a number

#behavioural6 - clear unsent events

Test steps:

  • Establish session
  • Start tracking
  • Move and click mouse
  • Stop tracking
  • Clear collected events
  • Send events
Expected result:
  • Event upload request is submitted with empty event
  • Refreshing session state shows score as a number

#behavioural7 - periodic sending

Test steps:

  • Establish session
  • Start tracking
  • Start periodic sending
  • Move and click mouse for a while (10-20) seconds
  • Stop periodic sending
Expected result:
  • Event upload requests are submitted periodically
  • Event upload request timestamps do not overlap and no same events are submitted twice
  • After stopping no events are sent
